What it is

A ReAct agent that treats its toolbox as a protocol, not a hardcode

Most agents wire each capability in by hand. agentic-ai instead exposes its database as a standard MCP server and gives the agent a single mcp tool — the agent learns what that server can do at query time via tools/list. Add a capability to the server and the agent picks it up with no code change.

🔁

ReAct reasoning loop

Reason → Plan → Act → Observe → repeat. The agent thinks, picks an action, executes a tool, reads the observation, and iterates until it has a grounded final answer.

🔍

Dynamic tool discovery

The agent holds a single mcp umbrella tool and discovers concrete operations at runtime via the standard MCP tools/list action — no per-operation pre-registration.

🔌

Dual LLM backend

Local Ollama by default; Anthropic Claude when an API key, model, and credit balance are all configured. Both implement one LLMCaller interface — swap with config, not code.

📐

Tool schema registry

Every tool owns its full definition (name, description, input_schema) through a Schema() method. The registry compiles these into the system prompt — descriptions live in exactly one place.

🧩

Standard MCP, no coupling

The MongoDB server speaks plain MCP over SSE. Any compatible client — Claude Desktop, Cursor, or a custom agent — connects with no special headers or auth.

🗂️

Multi-source grounding

Beyond the database, the agent reasons over local PDF vector search and live web results via the Tavily API — choosing the right source per step.

System design

How it all fits together

Two decoupled Go services and a pluggable LLM brain, joined by an open protocol. The agent reasons; the MCP server exposes data as discoverable tools; nothing is hardcoded between them.

The ReAct loop, end to end

A single endpoint runs the full reason → act → observe cycle, discovering and calling tools over MCP until the LLM produces a grounded final answer.

Agent side

Embeds the query in a dynamically-built ReAct prompt, selects the LLM backend from config, and orchestrates tool calls. It knows MCP only as a black box — one mcp tool whose operations it discovers, never hardcodes.

MCP server side

Opens an SSE stream, advertises its tool catalog via tools/list, and executes tools/call requests as MongoDB CRUD against the agentic_mcps database — returning structured CallToolResult payloads.
