⚙️  CONFIGURATION
────────────────────────────────────────────────────────────────────────────

config.json parameters:

OLLAMA_HOST          Where Ollama runs (default: http://localhost:11434)
OLLAMA_MODEL         LLM model to use (default: gemma4:e4b)
TAVILY_API_KEY       Web search API key (get from https://tavily.com)
MAX_STEPS            Max reasoning iterations (default: 10)
MAX_RETRIES          HTTP retry attempts (default: 3)
MAX_TOKENS           Token limit per LLM call (default: 1024)
SEARCH_ENDPOINT      Your PDF search API (default: http://localhost:8081/api/search)

Change any parameter without rebuilding—just restart the agent.


🎯 HOW IT WORKS
────────────────────────────────────────────────────────────────────────────

1. User asks a question
2. Agent runs ReAct loop (up to MAX_STEPS iterations)
3. Each step: Thought → Action → Observation
4. System prompt guides tool selection:
   • Searches PDF first (primary source)
   • Uses web search as fallback
5. Returns "Final Answer" when done or max steps hit
6. Graceful degradation: returns best answer so far (not an error)


🛡️  GUARDRAILS
────────────────────────────────────────────────────────────────────────────

✓ Max Steps        - Stops at MAX_STEPS, returns best answer
✓ Max Retries      - Retries HTTP calls up to MAX_RETRIES times
✓ Max Tokens       - Every LLM call respects MAX_TOKENS
✓ Timeout Safe     - Handles network/parsing errors gracefully

🚨 TROUBLESHOOTING
────────────────────────────────────────────────────────────────────────────

"Connection refused" on startup?
  → Check Ollama is running: ollama serve
  → Check PDF search API is running on port 8081

Slow responses?
  → Reduce MAX_STEPS
  → Lower MAX_TOKENS
  → Use a faster model: ollama pull mistral

Web search not working?
  → Check TAVILY_API_KEY is valid
  → Test: curl https://api.tavily.com/search?api_key=YOUR_KEY&query=test

Tool never called?
  → Check Ollama model supports ReAct reasoning
  → Verify system prompt is clear
  → Try a larger model: ollama pull neural-chat
